## 5. Known issues / action items (do not ship publicly without addressing)

1. **🚨 SECURITY — `/api/send-daily` is unauthenticated.**
   `CRON_SECRET` env var is **not set on Vercel**. Anyone hitting the URL can trigger a real fan-out email to every subscriber. Flagged by email-upgrade agent.
   **Fix:** set `CRON_SECRET` in Vercel Project → Settings → Environment Variables, then have `api/send-daily.js` reject requests whose `Authorization: Bearer <secret>` header doesn't match. Update Vercel cron config to send that header.
   **Blocking** any broader public launch.

2. **Amazon ASIN dictionary has placeholders / duplicates.**
   `amazon.js` has stub ASINs and duplicate entries (notably McCormick spices). The `asin-expansion` agent is fixing — wait for that branch.

3. **Stop & Shop / ShopRite chips broken on prod.**
   Both stores choke on multi-ingredient queries. The `fix-store-chips` agent built per-ingredient surgical search pages. Patch + drop-in files in `fix-store-chips-handoff/`.

4. **Daily email template is the old version on prod.**
   The richer template (eyebrow, primary CTA, per-dish ratings, calendar block, store-chip row) lives only in `email-upgrade-deliverable/email.js`. Has **not** been deployed.

5. **Local clone is in an inconsistent state.**
   `dinner-tonight-repo/` is on a stale `master` branch (`e1123d3`, 1 commit) that is FAR behind `origin/main`. It also has uncommitted changes from the chip-fixer flavor drop-in. **Do not push `master` to `main`** — it would delete 1,066 lines including `amazon.js`, `api/amazon-cart.js`, `api/walmart.js`, etc. Always work off `origin/main`.
