## Parser & infrastructure improvements shipped in this pass

1. **En-dash / em-dash normalization** — `parseOneIngredient` now replaces `–` and `—` with ASCII hyphen before the quantity regex. Fixes "4–6 chicken thighs" → "Chicken thighs" (previously parsed as a literal name "4–6 chicken thighs" that no store could find).
2. **Optional protein:/optional X: drop** — lines beginning `optional [noun]:` are now dropped explicitly (previously the "or" inside the suggestion list leaked into the search query).
3. **Container/portion-word stripping** — LEADING_UNIT_RE extended with `strips|balls|wedges|ears|ribs|knobs|chunks|cubes|rounds|rings|kernels`. Examples fixed: "4 strips bacon" → "Bacon", "1 ball fresh mozzarella" → "Mozzarella", "2 ears corn" → "Corn".
4. **Unicode-aware Walmart normalize** — `walmart.js#normalize()` now NFD-decomposes accents and strips combining marks. "Gruyère" now matches the "gruyere" key.
5. **Centralized synonyms layer** (`synonyms.js`, 177 entries) — every per-item / bulk search URL across Amazon, Walmart, FreshDirect, Stop & Shop, ShopRite, Instacart now canonicalizes ingredient names through one source of truth before encoding the query. Examples: "scallions" → "green onions", "coriander" → "cilantro", "stock" → "chicken broth", "ball mozzarella" → "fresh mozzarella".
6. **Walmart item-ID map expansion** — from 98 → 410 entries. Categories added: sandwich-category breads (sourdough, rye, brioche, hoagie rolls), deli proteins (turkey, roast beef, salami, prosciutto, pastrami, bacon), 12+ cheeses (cheddar, provolone, swiss, brie, feta, mascarpone, ricotta, gruyere, fontina, goat cheese), fresh herbs (parsley, cilantro, thyme, dill, oregano, scallions), 20+ dried spices, pasta variants (gnocchi, orzo, farfalle, rigatoni), produce (arugula, broccoli, broccolini, cauliflower, brussels sprouts, jalapeno, mushrooms, ginger, avocado, baby potatoes), berries, sweeteners (honey, maple syrup, brown/powdered sugar), stocks/broths, condiments (soy sauce, sriracha, fish sauce, hoisin, mirin, balsamic, rice vinegar, dijon, mayo, pesto, tahini), tofu, deli grains (quinoa, farro, couscous, lentils).
7. **mailto:/sms: overflow truncation** — both URI builders now measure the encoded URI length and progressively drop tail ingredients (replacing them with "... and N more") when the limit (1900 chars for mailto, 240 for sms) would be exceeded. This prevents iOS Messages silently swallowing long shopping lists and Gmail clipping mailto links.

## Endpoint-level guarantees

Every store endpoint was invoked against every recipe (588 calls) and verified for:

1. `HTTP 200` (or `302` redirect with a valid `Location` header for Walmart);
2. response body contains the expected per-store branding + per-item search rows;
3. no `?q=`/`&k=`/`searchParams=` URLs with an empty query string;
4. for recipe-card: response body contains `<script type="application/ld+json">`; the JSON parses to a Recipe with `@type === "Recipe"` and `recipeIngredient` is a non-empty array of strings (Bring! requirement).

## Notes / open items

- Walmart item-IDs are best-effort: the cart-add URL falls back to the Walmart product search if Walmart returns "out of stock" for any item. Item-ID rot is monitored by failing gracefully (`lookupItemId` returns null → fallback search URL). When a future audit shows a specific item silently 404'ing on Walmart, just remove the line from `WALMART_ITEM_MAP` and the next request will use search instead.
- Stop & Shop and ShopRite per-item search URLs were verified locally only; live verification requires a Long Island ZIP code in the storefront cookie.
- Amazon Fresh strategy intentionally stays "per-item search" (no ASIN map). This is correct — the prior ASIN dictionary was the source of silent broken-cart bugs and ASINs rotate too quickly to keep current.
- The grocery-wiring agent could not directly read another agent's session transcript or send dispatch messages in this environment. The parser / synonyms / item-map work shipped is compatible with any recipe-expansion changes that follow the existing `{course, cat, ing, ...}` shape in `recipes.js`.
